Schema Architecture

The Universal Primitives

The 5 fundamental building blocks of an Agentic CMS. By using strict Enums and References, the AI generates semantic payloads, not unstructured HTML.

¶

Prose Block

type: document

Rich text container for narrative copy. Handles headings, body paragraphs, and stylistic overrides like pull quotes.

// Core Schema Fields
content_body: RichText
// Dictates visual rendering
semantic_variant: Enum
└─ [standard_body, lede_paragraph, pull_quote]
🖼️

Media Block

type: document

Visual asset container for images, looping videos, or AI-generated data visualizations.

media_type: Enum
└─ [static_image, looping_video, data_chart]
asset_reference: MediaLink
alt_text: String (Required)
aspect_ratio: Enum
└─ [16:9, 4:3, 1:1, auto]
👆

Action Block

type: document

Interactive trigger for conversions, navigation, or lead capture (buttons, links, micro-forms).

interaction_type: Enum
└─ [button, text_link, email_form]
action_label: String (Max 30 chars)
destination_url: URL
visual_hierarchy: Enum
└─ [primary, secondary, ghost]
🧭

Wayfinding Block

type: document

Structural navigation helpers like Table of Contents, Breadcrumbs, or Progress Wizards.

wayfinding_type: Enum
└─ [toc, breadcrumbs, wizard]
navigation_nodes: Array[Object]
├─ label: String
└─ target: String
scroll_behavior: Enum
Compositional! 🧩
🗂️

Card Block

type: document

A standardized micro-container for displaying a single entity. Notice how it references other blocks rather than redefining fields, keeping the AI's payload clean.

// Uses content References
title: String (Max 80 chars)
summary_text: String
meta_tag: String
🔗 thumbnail_media: Reference[Media Block]
🔗 card_action: Reference[Action Block]

Why this matters for the AI

By limiting the CMS to these 5 primitives, the Agent doesn't have to learn CSS, HTML, or proprietary site builders. It simply receives an Intent, selects a Layout Blueprint, and writes JSON objects containing these Primitives to fill the slots. The front-end application handles the rest.
