Here is a breakdown of the UX psychology and Information Architecture (IA) reasoning behind why each layout is perfectly suited to its underlying intent. 

By hardcoding these rules into the Agentic CMS, the AI doesn't have to "guess" at design—it relies on decades of proven human-computer interaction principles.

---

### 1. Direct Action (Convert) ➔ The Funnel Layout
* **The Psychology:** Hick’s Law (The time it takes to make a decision increases with the number and complexity of choices).
* **The Reasoning:** When the goal is conversion, the biggest enemy is distraction. 
    * **The Hero Slot (50/50):** The split design places the value proposition on the left (where Western readers look first) and the product validation (image/video) on the right. 
    * **The Social Proof Slot:** Immediately answers the user's subconscious objection: *"Can I trust you?"* Placing logos or stars right above the fold validates the hero claim.
    * **No Global Navigation (Optional):** In strict funnel layouts, the AI knows to strip away the top navigation bar. There should be only two ways off the page: click the CTA, or close the tab.

### 2. Deep Dive (Educate) ➔ The Editorial Layout
* **The Psychology:** Cognitive Pacing and the F-Pattern of reading.
* **The Reasoning:** Reading long-form content on a screen is physically fatiguing. The layout must artificially pace the reader.
    * **Asymmetrical 70/30 Split:** Typography rules dictate that a line of text should be no wider than 60–80 characters. If text spans the full width of a modern monitor, the eye gets lost returning to the next line. The 70% column constrains the text to the perfect reading width.
    * **The 30% Sidebar:** Acts as a persistent anchor. It holds the Table of Contents or related links, allowing users to jump around without having to scroll all the way back to the top.

### 3. Crossroads (Discover) ➔ The Hub Layout
* **The Psychology:** Information Foraging Theory (Users hunt for information the same way animals hunt for food; they follow "scent" trails).
* **The Reasoning:** Users arriving here don't have a specific target; they are browsing. 
    * **The Utility Slot:** Placed immediately below the hero, this offers search and filters. It gives control to the user to narrow the field if they feel overwhelmed.
    * **The Matrix Slot (Cards):** This relies on *Pattern Recognition*. By forcing every piece of content into an identical card container (Image + Title + Category + Link), the brain stops looking at the layout and only looks at the *differences* between the cards. It allows a user to scan dozens of items in seconds.

### 4. Resolution (Support) ➔ The Diagnostic Layout
* **The Psychology:** High-Anxiety Utilitarianism and Progressive Disclosure.
* **The Reasoning:** A user looking for support is usually frustrated. Marketing fluff, complex graphics, and animations will only make them angrier.
    * **The Utility Hero:** Massive search bar. The AI knows to put the solution engine front and center, bypassing any promotional copy.
    * **The Accordion/Step Slot:** Uses *Progressive Disclosure*. If you show a user a wall of text containing 50 troubleshooting steps, cognitive overload triggers a bounce. Accordions hide the complexity until the user explicitly asks for it by clicking.
    * **The Escalation Slot:** A highly visible "Contact Human" block at the bottom. Psychologically, knowing an escape hatch exists makes users more willing to try the automated troubleshooting first.

### 5. Evaluation (Compare) ➔ The Matrix Layout
* **The Psychology:** Choice Architecture and the Anchoring Effect.
* **The Reasoning:** The human brain is terrible at holding complex data points in short-term memory. It needs side-by-side juxtaposition to weigh trade-offs.
    * **The Comparison Grid:** Places options vertically side-by-side. The AI leverages the *Center Stage Effect* by slightly enlarging or highlighting the middle/recommended tier, subconsciously guiding the user toward the highest-value option.
    * **The Feature Checklist:** Uses sticky column headers. As the user scrolls down through 40 different feature checkmarks, they never have to guess which column belongs to which tier, preventing memory strain.

### 6. Operations (Manage) ➔ The Control Room Layout
* **The Psychology:** Situational Awareness and Spatial Memory.
* **The Reasoning:** Logged-in users (admins, patients, buyers) treat these pages like software tools, not marketing documents. They rely on muscle memory to perform repetitive tasks.
    * **Persistent Nav Slot:** The sidebar never moves. This gives the user a permanent anchor point, ensuring they never feel "lost" inside the application.
    * **The Summary Slot:** Provides an instant "pulse check" (e.g., total revenue, unread alerts) so the user doesn't have to dig for the most critical metrics.
    * **Modular Dashboard Slot:** Uses masonry or distinct bounding boxes. The AI knows to put critical alerts top-left, and historical data bottom-right, respecting how the eye naturally falls across a control panel.
